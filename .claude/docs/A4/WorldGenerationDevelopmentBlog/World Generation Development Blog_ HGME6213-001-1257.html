Features

Noise Layers

  • Use the same noise layers as Minecraft: Continentalness, Erosion, Weirdness, Peaks and Valleys, Humidity, and Temperature.

Terrain

  • Generate terrain using 3D density noise with possible top and bottom slides.
  • Shape terrain using curves based on Continentalness, Erosion, and Peaks and Valleys.

Biomes

  • Generate biomes using lookup tables with Continentalness, Erosion, Peaks and Valleys, Humidity, and Temperature.

  • Map biomes to a limited subset for which assets exist.

Caves

  • Implement cheese caves.

  • Implement spaghetti caves.

Surface Replacement

  • Use a simplified version of Minecraft’s surface replacement system.

Vegetal Decoration

  • Include only trees for vegetation.

Carvers

  • Use carvers for caves, canyons, and rivers.
  • This part I am least certain about and will require investigation and research when I get to this.

Terrain Density and Bias

October 15, 2025

I now have some 3D density noise with bias to give some starting terrain to shape.

Raw Density Noise

Biased Density Noise

Calculation

For each block, calculate a density value. 

D(x, y, z) = N(x, y, z, s) + B(z)

where:

N(x, y , z , s) = 3D Perlin noise
B(z) = b × (z − t)

s – Perlin noise scale
o – Perlin noise number of octaves
t – default terrain height
b – density bias per block delta from default terrain height

Density is in the range (-1, 1) with negative being more dense.  After calculating the density set the block types to stone if D < 0, or air otherwise.

Continents

October 17, 2025

I have now shaped the terrain with continents, or continentalness.

Initial Terrain

Terrain with just depth noise and bias, no continents. 

Terrain with Height Offset

Terrain after continent height offset.

Terrain with Squashing and Height Offset

Terrain after continent squashing. Squashing means to scale numbers in around a specific pivot, which in our case in the terrain height after offset.

Calculations

  • Have two curves, one for continent height offset and another for continent squashing factor.
  • Compute 2D Perlin noise (per-column) for a continents map. This will be the input to the curves.
  • Get the value from the height offset curve, h, and subtract that from density.
  • Get the value from the squashing curve, s, multiply by a terrain offset, t, and add that to density.

b = default_terrain_height + (h * (chunk_size_z / 2))
t = (z - b) / b
density -= h
density += s * t

where:

z –  block z-coordinate
b – base terrain height
t – height offset from base terrain height
h – output from the height offset curve
s – output from the squasing curve

Surface Replacement

October 21, 2025

I have used the biomes that I generated in my previous change to do surface replacement. Much of the table lookup I did with if and switch statements. It was tedious and not pretty but got the job done. I would have liked a table lookup system of some sort but didn't have the data in a format that was readily usable.

Surface replacement scans downward to locate the first solid block, then assigns surface and subsurface materials based on biome type. Sand appears in beaches and deserts, grass or snow in plains and cold biomes, and dirt or stone below. Noise functions vary the depth of dirt, lava, and obsidian layers, adding natural irregularity. The system replaces stone and air with proper surface, dirt, or ore layers. Water fills air blocks below sea level.

Which is admittedly more than just surface replacement so maybe I could have picked a better name.

Process

  1. Determine the surface height of each vertical column.
  2. Assign surface and subsurface materials by biome.
  3. Vary dirt, lava, and obsidian depth with noise.
  4. Replace stone blocks to create surface, dirt, and ore.
  5. Fill air blocks below sea level with water.Threaded mesh generation

Vegetal Decoration

October 21, 2025

I have initial tree generation based on biome, temperature, and humidity. Currently I have trees of 1, 2, or 3 blocks and will need to spend some time making the rest of the stamps. 

I plan to use this image of trees to build out the stamps.

Tree generation decides if and what kind of tree grows in each column using noise-based thresholds. Only the highest noise value in a small area produces a tree, preventing overlap. Biome, temperature, and humidity determine tree type and size. Deserts spawn cactus, forests spawn oak or birch, jungles spawn jungle trees, and cold biomes spawn spruce or snowy spruce. Temperature and humidity slightly adjust thresholds for tree appearance. Trees are placed using tree stamps. A tree stamp is a collection of offsets and block types that defines the structure of the tree.

Process

  1. Use noise to decide where trees grow.
  2. Only generate trees only on tiles that have a higher noise value that all its neighbors.
  3. Generate small, medium, and large trees.
  4. Use threshholds to decide whether to generate a tree and of what size.
  5. Set initial threshholds based on biome.
  6. Adjust thresholds slightly using temperature and humidity values.
  7. Choose tree type based on biome.
  8. Generate trees above sea level on valid surface blocks.
  9. Build trees using tree stamps.
    1. A stamp is a vector of coordinate offset and block type pairs.
    2. Build them during world startup.
    3. Iterate over the vector and at each offset change the block type.
  10. IMPORTANT
    1. Do not not change block types outside your chunk.
    2. Check for trees outside of your chunk based on a maximum tree radius.
    3. Iterate over the entire stamp, just skip blocks not in your chunk.
